# 激励计划数据摘要

获取线上外显的激励计划相关数据，包含在线激励计划列表、计划高收益博文示例、博主近7日优质博文及命中计划。

---

## 获取激励计划数据摘要

使用 `adincentive-summary` 命令获取激励计划数据摘要。

### 基本用法

```bash
node scripts/weibo-skill.js adincentive-summary
```

### 参数说明

无额外参数，Token 由脚本自动获取。

### 返回结果

返回的 `data` 字段为 `AdIncentiveSummary` 对象，包含以下维度：

#### `data.uid` — 用户 uid

#### `data.userLevel` — 用户认证等级

| 字段 | 类型 | 说明 |
|------|------|------|
| `userLevel` | string | 用户认证等级（如 "金V"、"蓝V"、"普通用户" 等） |

> `userLevel` 反映博主的认证身份，不同激励计划可能面向不同认证等级的创作者，推荐计划时需结合此字段判断博主是否符合计划的面向对象。

#### `data.plans` — 在线激励计划列表（来自 plans 接口）

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 计划名称 |
| `desc` | string | 计划描述/规则简介 |
| `category` | string | 业务分类标识（hot/video/campus/cross/other） |
| `categoryName` | string | 业务分类中文名 |
| `prUrl` | string | 计划PR博文链接（详细了解规则的博文地址） |
| `hasRank` | boolean | 是否有高收益博文示例（周榜数据） |
| `tags` | array | 激励内容方向标签列表（PlanTag[]） |

**`data.plans[].tags[]`** — 激励内容方向标签

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 方向标签名称 |

#### `data.totalPlans` — 在线激励计划总数

| 字段 | 类型 | 说明 |
|------|------|------|
| `totalPlans` | integer | 当前在线且在前端展示的激励计划总数 |

> 仅提供前端【展示】且【在线】的计划数据。

#### `data.planRanks` — 计划高收益博文示例（来自 planrank 接口）

Map 结构，key 为整数标识，value 为该计划的榜单数据 `PlanRank` 对象。通过 `planName` 与 `plans` 中的计划关联。

**`data.planRanks[<key>]`** — 单个计划的榜单数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `planName` | string | 计划名称 |
| `rankRule` | string | 高收益博文示例规则说明 |
| `prUrl` | string | 计划PR博文链接 |
| `period` | string | 当前示例周期（最新一周） |
| `categories` | array | 分类tab列表（RankCategory[]） |
| `blogs` | array | 高收益博文示例列表（RankBlog[]） |
| `totalInPage` | integer | 当前页博文数量 |
| `hasMore` | boolean | 是否有更多数据 |

**`data.planRanks[<key>].categories[]`** — 榜单分类tab

| 字段 | 类型 | 说明 |
|------|------|------|
| `cateId` | string | 分类ID |
| `name` | string | 分类名称 |

**`data.planRanks[<key>].blogs[]`** — 高收益博文示例

| 字段 | 类型 | 说明 |
|------|------|------|
| `mid` | string | 博文mid |
| `text` | string | 博文正文内容 |
| `topics` | array | 命中话题列表（string[]） |
| `category` | string | 所属类别/tab名称（若有） |
| `hasVideo` | boolean | 是否含视频 |

> 对用户无需强调是"周榜"，对用户称"计划高收益博文示例"即可。仅提供最新周数据。上榜博文信息包括博文内容、所属类别（tab，若有），**不给金额**。

#### `data.myBlogs` — 博主近7日高收益博文列表（来自 myblogs 接口）

| 字段 | 类型 | 说明 |
|------|------|------|
| `mid` | string | 博文mid |
| `text` | string | 博文正文内容 |
| `createTime` | string | 发博日期（格式：yyyy-MM-dd） |
| `topics` | array | 命中话题列表（string[]） |
| `hasVideo` | boolean | 是否含视频 |
| `matchedPlans` | array | 命中的激励计划列表（MatchedPlan[]） |

**`data.myBlogs[].matchedPlans[]`** — 博文命中的激励计划

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 计划名称 |
| `prUrl` | string | 计划PR链接 |

> 近7日高收益博文及命中的计划，**不给金额**。

### 使用示例

```bash
node scripts/weibo-skill.js adincentive-summary
```

### 注意事项

1. 需要先执行 `login` 命令完成配置
2. 仅提供前端【展示】且【在线】的计划数据，已下线或未展示的计划不会返回
3. `planRanks` 中仅包含 `hasRank` 为 `true` 的计划的榜单数据
4. 高收益博文示例仅提供最新一周数据，不提供历史周数据
5. 上榜博文信息和博主优质博文均**不包含金额数据**
6. `myBlogs` 为近7日数据，超过7天的博文不会返回
7. `planRanks` 为 Map 结构，通过 `planName` 与 `plans` 中的计划关联

## 数据分析能力

本技能在基础数据之上，提供激励计划参与建议分析能力。当用户询问如何参与激励计划、哪些计划适合自己、如何获得更多创作激励等问题时，综合运用以下能力输出分析报告。

> ⚠️ **分析报告必须遵守以下规则**：
> 1. **报告中提到的每个计划必须附带 PR 链接**（`prUrl`），方便用户点击查看详细规则。没有 PR 链接的计划也需标注"暂无PR链接"。**推荐计划不使用表格展示，直接使用 Markdown 链接格式** `[url](url)` 输出。
> 2. **分析必须参考博主自己的发博数据**。在输出参与建议前，必须先通过 `status` 命令获取博主的历史发博数据，结合博主的内容方向、发博频率、互动数据等进行分析，不可脱离博主实际情况泛泛而谈。

### 一、数据整理

从接口返回数据中整理以下关键信息，作为分析基础。其中 PR 博文链接内容和博主历史发博数据需要通过其他功能获取。

#### 在线计划信息

| 数据项 | 数据来源 | 说明 |
|--------|----------|------|
| 计划名称 | `plans[].name` | 如"热点优质创作计划" |
| 计划描述 | `plans[].desc` | 计划规则简介 |
| PR博文链接 | `plans[].prUrl` | 详细了解规则的博文地址 |
| 内容方向标签 | `plans[].tags[].name` | 计划激励的内容方向 |
| 业务分类 | `plans[].categoryName` | 如"热点"、"视频"等 |
| 是否有榜单 | `plans[].hasRank` | 是否有高收益博文示例 |

#### 高收益博文示例信息（若有榜单）

| 数据项 | 数据来源 | 说明 |
|--------|----------|------|
| 榜单规则 | `planRanks[<key>].rankRule` | 参与规则说明 |
| 示例周期 | `planRanks[<key>].period` | 最新一周 |
| 分类tab | `planRanks[<key>].categories[]` | 可筛选的分类 |
| 上榜博文 | `planRanks[<key>].blogs[]` | 高收益博文示例 |
| 上榜博文话题 | `planRanks[<key>].blogs[].topics[]` | 命中的话题 |
| 上榜博文类别 | `planRanks[<key>].blogs[].category` | 所属类别/tab |
| 上榜博文是否含视频 | `planRanks[<key>].blogs[].hasVideo` | 是否含视频 |

#### 博主优质博文信息

| 数据项 | 数据来源 | 说明 |
|--------|----------|------|
| 博文内容 | `myBlogs[].text` | 博文正文 |
| 发博日期 | `myBlogs[].createTime` | 发博时间 |
| 命中话题 | `myBlogs[].topics[]` | 博文命中的话题 |
| 是否含视频 | `myBlogs[].hasVideo` | 是否含视频 |
| 命中计划 | `myBlogs[].matchedPlans[]` | 博文已命中的激励计划 |

#### PR博文链接内容获取

每个激励计划的 `prUrl` 是该计划的 PR 博文链接，包含计划的详细规则、参与方式、激励面向对象、激励标准等关键信息。需要通过 `status-show` 命令获取 PR 博文的具体内容。

> ⚠️ **PR 博文内容中通常包含激励面向对象信息**（如"面向金V/橙V创作者"、"面向自媒体博主"等），推荐计划时必须结合 `userLevel` 判断博主是否符合该计划的面向对象，不符合面向对象的计划不应推荐。

**获取方式**：

```bash
node scripts/weibo-skill.js status-show --url=<prUrl>
```

| 数据项 | 数据来源 | 说明 |
|--------|----------|------|
| PR博文正文 | `status-show` 返回的 `text` | 计划的详细规则、参与方式、激励标准等 |
| PR博文互动数据 | `status-show` 返回的 `reposts_count`/`comments_count`/`attitudes_count` | 了解该 PR 博文的传播情况 |

> PR 博文正文是了解计划详细规则的最直接来源，建议在分析时优先获取并阅读 PR 博文内容，以补充 `desc` 字段的简要描述。

#### 博主历史发博数据获取

当需要更全面地了解博主发博情况时，应调用 `status` 命令获取用户的历史发博数据，结合激励计划数据进行更深入的分析。

**获取方式**：

```bash
node scripts/weibo-skill.js status
```

| 分析维度 | 数据来源 | 说明 |
|----------|----------|------|
| 博主历史发博内容 | `status` | 了解博主长期内容方向，更精准匹配计划 |
| 博主发博频率 | `status` | 评估博主能否持续参与计划 |
| 博主互动数据 | `status` | 评估博文质量，判断能否获得高收益 |
| 博主内容与计划匹配度 | `status` + `adincentive-summary` | 综合判断最适合的计划 |

> 使用 `status` 获取发博数据后，与激励计划数据交叉分析，可提供更精准的参与建议。

### 二、激励计划参与建议分析

若用户询问激励计划参与建议、如何获得更多激励等问题，请按以下步骤进行分析：

**第一步：获取博主发博数据（必须）**

在进行分析前，**必须**先通过 `status` 命令获取博主的历史发博数据，了解博主的内容方向、发博频率、互动情况。没有博主发博数据，无法给出有针对性的建议。

**第二步：确认博主认证等级与计划面向对象**

从 `userLevel` 获取博主当前认证等级（如"金V"、"蓝V"、"普通用户"等）。获取各计划的 PR 博文内容，确认每个计划的**激励面向对象**（如"面向金V/橙V创作者"、"面向自媒体博主"等），判断博主是否符合各计划的参与资格。不符合面向对象的计划不应推荐。

**第三步：梳理当前在线激励计划**

从 `plans` 中整理所有在线计划，提取每个计划的名称、描述、分类、内容方向标签、**PR链接**。若有榜单（`hasRank` 为 `true`），标注该计划有高收益博文示例可参考。

> ⚠️ 报告中每个计划**必须附带 PR 链接**（`prUrl`），方便用户点击查看详细规则。推荐计划不使用表格展示。

**第四步：分析博主已有博文命中情况**

从 `myBlogs` 中分析：
- 博主近7日有哪些博文已命中激励计划（`matchedPlans` 非空）
- 命中了哪些计划，分别命中了什么话题
- 哪些博文尚未命中任何计划但内容方向可能适合

同时结合 `status` 获取的博主历史发博数据，分析博主长期内容方向与计划的匹配度。

**第五步：匹配适合博主参与的计划**

结合以下信息进行匹配：

| 匹配维度 | 数据来源 | 匹配方法 |
|----------|----------|----------|
| **博主认证等级与计划面向对象** | `userLevel` + PR博文内容中的面向对象说明 | 博主认证等级不符合计划面向对象的，不应推荐 |
| 博主发博内容方向 | `myBlogs[].text` + `myBlogs[].topics` + `status` 历史发博 | 与计划内容方向标签 `plans[].tags` 对比 |
| 博主是否发视频 | `myBlogs[].hasVideo` + `status` 历史发博 | 视频类计划（`category` 为 `video`）优先推荐给有视频创作能力的博主 |
| 计划榜单话题 | `planRanks[<key>].blogs[].topics[]` | 博主内容与榜单话题重合度高的计划优先推荐 |
| 博主未命中计划 | `myBlogs[].matchedPlans` 为空的博文 | 找出内容可调整以命中计划的博文 |
| 博主发博频率 | `status` | 评估博主能否持续参与计划 |
| 博主互动数据 | `status` | 评估博文质量，判断能否获得高收益 |

> ⚠️ **认证等级与面向对象匹配是首要筛选条件**。如果博主认证等级不符合计划的面向对象，即使内容方向匹配也不应推荐，但可在报告中提及该计划供博主了解，并说明升级认证后可参与。

**第六步：参考高收益博文示例**

对有榜单的计划，从 `planRanks` 中提取：
- 榜单规则（`rankRule`）：告知用户参与方式和激励规则
- 上榜博文示例（`blogs`）：分析上榜博文的内容特征、话题使用、是否含视频等
- 分类tab（`categories`）：告知用户可参与的内容分类

**第七步：输出分析报告**

报告包含：推荐参与的计划及理由、发博内容建议、参与计划的发博规划。

#### 报告结构

> ⚠️ 报告中每个计划**必须附带 PR 链接**（`prUrl`），方便用户点击查看详细规则。没有 PR 链接的计划标注"暂无PR链接"。推荐计划不使用表格展示，直接使用 Markdown 链接格式 `[url](url)`。

```
🎯 激励计划参与建议报告

一、博主基本信息与发博数据分析
  认证等级：[userLevel]
  面向对象匹配：[根据PR博文内容判断博主符合哪些计划的面向对象]
  [基于 status 获取的博主历史发博数据分析]
  - 内容方向：[博主主要发博方向]
  - 发博频率：[日均/周均发博数]
  - 视频创作能力：[是否经常发视频]
  - 互动表现：[博文互动情况]

二、当前在线激励计划概览
  共 X 个在线计划：
  - [计划名称]（[分类]）：[描述]
    PR链接：[prUrl](prUrl)（若无则标注"暂无PR链接"）
  - ...

三、博主近7日博文命中情况
  已命中计划的博文：
  - [博文摘要]... → 命中 [计划名称]（话题：#xxx#）

  未命中计划的博文：
  - [博文摘要]... → 未命中任何计划

四、推荐参与的计划（结合博主发博数据）
  ⭐ [计划名称]
  推荐理由：[结合博主内容方向、发博频率、互动数据与计划匹配度说明]
  PR链接：[prUrl](prUrl)
  参与方式：[根据计划描述和榜单规则说明]
  内容建议：[根据计划标签和榜单话题给出具体发博建议]

五、发博规划
  近期可执行的发博计划：
  1. [具体发博建议，包含话题、内容方向、是否需要视频等]
  2. ...
```

### 三、认证升级引导

> ⚠️ **当博主认证等级低于金V或橙V时，必须在报告末尾增加升级引导和追问**。

#### 适用场景

根据 `userLevel` 判断博主认证等级，以下情况需要增加升级引导：

| 当前等级 | 升级目标 | 是否需要引导 |
|----------|----------|--------------|
| 普通用户 | 黄V → 橙V → 金V | ✅ 需要 |
| 黄V | 橙V → 金V | ✅ 需要 |
| 橙V | 金V | ✅ 需要 |
| 金V | 已达最高等级 | ❌ 不需要 |

#### 引导内容

在报告末尾添加追问，询问用户是否需要升级规划：

```
---
📌 作为[当前等级]，部分高收益激励计划暂时无法参与。如果你希望解锁更多计划，我可以帮助你规划升级路径。需要我帮你分析当前创作数据并制定升级计划吗？
```

#### 升级规划服务对接

当用户回复"需要"或表示希望获取升级规划时，应调用 `creator-summary` 命令获取创作者数据，并按照"金橙V 升级分析"能力（详见 [weibo-creator.md](./weibo-creator.md)）输出升级规划报告。
